# PepsiCo Pulse BI - دليل النشر والتشغيل

## الملف اللي تنشره
**`PepsiCo_Pulse_BI.html`** - ملف واحد مكتفٍ بذاته (self-contained) فيه كل حاجة:
الداتا + الرسومات + الفلاتر + الصور. مفيش سيرفر ولا قاعدة بيانات مطلوبة.

> بيانات الدخول التجريبية: **moataz / moataz**

---

## 1) النشر للناس - 3 طرق (الأسهل أولاً)

### أ) Netlify Drop - أسرع طريقة (دقيقة واحدة، مجاني)
1. افتح <https://app.netlify.com/drop>
2. اسحب ملف `PepsiCo_Pulse_BI.html` وارميه في الصفحة.
3. هيطلعلك لينك زي `https://random-name.netlify.app` - ابعته لأي حد.
4. (اختياري) من Site settings تقدر تغيّر الاسم أو تربط دومين خاص.

> نصيحة: سمّي الملف `index.html` قبل ما ترفعه علشان اللينك يفتح عليه مباشرة من غير اسم الملف.

### ب) GitHub Pages - لينك دائم مجاني
1. اعمل repository جديد على GitHub (Public).
2. ارفع الملف باسم `index.html`.
3. Settings → Pages → Source = `main` / root → Save.
4. بعد دقيقة هيشتغل على `https://<username>.github.io/<repo>/`.

### ج) سيرفر الشركة الداخلي (SharePoint / IIS / أي Web server)
- **SharePoint:** ارفع الملف لمكتبة مستندات وافتحه عبر "View in browser" (أو استخدم
  صفحة Embed). قد تحتاج صلاحية تشغيل HTML من الـIT.
- **IIS / Apache / Nginx:** انسخ الملف لمجلد الموقع (مثلاً `wwwroot`) وافتحه عبر الـURL.
- ميزة الطريقة دي: يفضل داخل الشبكة ومحمي بصلاحيات الشركة.

> الملف **static** بالكامل - أي استضافة ملفات ثابتة تشغّله. مفيش backend.

### مشاركة سريعة من غير استضافة
تقدر تبعت الملف نفسه بالإيميل/تيمز؛ المستقبِل يفتحه بالدبل-كليك في أي متصفح.
(عيبه: حجمه ~2.3MB وبيتفتح نسخة مستقلة عند كل واحد.)

## 2) إيه اللي اتعمل في النسخة دي
- **داتا حقيقية متنوعة 3 سنين (2024-2026):** 23,040 صف مبيعات شهرية، بـseasonality
  (المشروبات تعلى صيفاً، السناكس آخر السنة)، نمو سنوي ~12.5%، 8 مناطق EMEA،
  4 قنوات بيع، 20 منتج (مشروبات + سناكس + أطعمة + مشروبات طاقة).
- **سلايسرز/فلاتر حيّة** فوق كل داشبورد: السنة / المنطقة / القناة / الفئة / العلامة /
  المنتج - متعددة الاختيار، وكل الرسومات والـKPIs بتتحسب لحظيًا.
- **YoY حقيقي** (لأن فيه أكتر من سنة) في الكروت والمنتجات.
- خدمة/تسليم/مخزون بتتأثر بالمنطقة المختارة، والعمالة بتتأثر بالسنة.
- **مساعد ذكي (NLP) بالعربي والإنجليزي** بدل "البحث بالأرقام" القديم - بيفهم
  المنتجات/العلامات/المناطق/القنوات/الفئات/السنين، وبيعمل: استخراج أي رقم،
  ترتيب (top/bottom)، مقارنات (Pepsi vs 7UP)، اتجاهات، **وتحريك الداشبورد بالأمر**
  (مثلاً «اعرض الأونلاين في مصر» بيفلتر كل الشاشات). السيرش فوق بيوصّل له برضه.

## 3) الملفات في المشروع
| الملف | الوصف |
|---|---|
| `PepsiCo_Pulse_BI.html` | **الناتج النهائي - ده اللي تنشره** |
| `gen.js` | مولّد الداتا (الأبعاد + التوزيعات + الـseasonality) |
| `portal_logic.js` | منطق التطبيق: محرّك التجميع + الفلاتر + الرسم |
| `build.js` | يدمج الأصول + الداتا + المنطق وينتج الـHTML |
| `export_csv.js` | يصدّر الداتا CSV تتفتح في Excel |
| `data_export/*.csv` | الداتا الجديدة كجداول (بديل الإكسيل القديم) |
| `test_compute.js`, `render_smoke.js` | اختبارات تحقق |

## 4) تعديل الداتا أو إعادة التوليد
```bash
# تعديل المنتجات/المناطق/التوزيعات: عدّل gen.js
node gen.js          # يطبع تحقق سريع للأرقام
node build.js        # يعيد بناء PepsiCo_Pulse_BI.html
node export_csv.js   # (اختياري) يعيد تصدير الـCSV

# اختبارات
node test_compute.js   # أرقام التجميع تحت كل فلتر
node render_smoke.js    # تشغيل كل الداشبوردات بدون أخطاء
```
> عايز تضيف داتا أكتر؟ زوّد المنتجات في `gen.js` (مصفوفة `PRODUCTS`) أو المناطق
> (`REGIONS`) وأعد `node build.js`. عايز تغيّر سنين؟ عدّل `YEARS`.
